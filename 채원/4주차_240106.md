## 운영체제 면접 질문 리스트

<h3>기본 개념</h3>

- 운영체제란 무엇인가요? 어떤 일을 하나요?
    - 운영체제란, 컴퓨터의 하드웨어와 소프트웨어간의 효율적인 상호작용을 관리하기 위한 일종의 소프트웨어입니다. 
    - CPU나 메모리와 같은 시스템 리소스를 효율적으로 관리하여 여러 응용프로그램이 동시에 실행될 수 있게 만듭니다. 
    - 그 외에도, 프로세스나 파일 시스템을 관리하거나 사용자에게 CLI와 같은 인터페이슨를 제공합니다.
- CPU의 원리를 간단히 설명해주세요.
    - (작성중)
- 운영체제의 커널이란 무엇인가요? 어떤 예시가 있나요?
    - 운영체제 또한 응용 프로그램이기 때문에, 메모리 위에 올라가있어야만 실행될 수 있습니다. 하지만, 운영체제는 그 크기가 너무 커, 항상 메모리 위에 둘 순 없습니다.
    - 따라서 반드시 필요한 부분만을 메모리 위에 올려두는데, 그 영역을 커널 영역이라고 합니다.
- 운영체제의 시스템 호출이란 무엇인가요? 또한 어떤 종류가 있나요?
    - 사용자 모드에서 실행되는 응용 프로그램이 운영체제의 서비스를 요청하는 인터페이스를 의미합니다. 
    - 응용 프로그램은 시스템 호출을 통해 커널 모드로 전환된 운영체제를 통해 원하는 자원으로의 접근 등 원하는 요청을 실행합니다.
    - 주로, fork(), exit() 등과 같은 프로세스 제어, open(), write() 등과 같은 파일 관리, chmod, getpid 등과 같은 것이 있습니다.
- 운영체제 내에서 시스템 호출이 일어나면 어떤 일이 일어나는지 순서대로 설명해주세요.
    - 예를 들어, 한 텍스트 파일을 여는 소스코드가 있다고 가정한다면, 
    - 소스코드를 실행한 뒤 -> open()이라는 라이브러리가 import 되고, 실행되는데 이 때, open()이라는 라이브러리가 실행되기 위해 운영체제의 커널 모드로 전환되고, 운영체제가 시스템 자원을 이용하기 위해 시스템 호출을 보내고, 운영체제가 실행한 결과를 다시 사용자 모드로 돌아가서 리턴합니다. 
- 스왑(Swap)이란 무엇인가요? 그리고 어떤 상황에서 사용되나요?
    ー 스왑이란, 컴퓨터 시스템에서 메모리의 일부를 디스크로 옮기는 기술을 의미합니다. 이를 통해, RAM의 물리적인 한계를 어느정도 극복할 수 있습니다. 
    - 스왑은 주로, 물리적인 RAM이 부족하거나, 페이지 폴트 현상이 일어났거나, 프로세스가 종료되는 등 메모리를 효율적으로 관리할 필요가 있을 때 행해집니다. 
- 컨텍스트 스위칭이란 무엇인가요? 그리고 이 과정에서 어떤 일이 일어나나요?
    - 여러 프로세스 혹은 스레드를 처리해야 할 때, 현재 진행중인 task의 상태를 PCB에 저장하고, 다음에 진행할 task의 PCB 데이터를 읽어 CPU에서 실행될 수 있도록 레지스터에 적재하는 과정을 의미합니다. 
    - 주로, 여러 task(프로세스나 스레드)가 동시에 실행될 수 있도록 하기 위해 행해지는데, 예를 들어, I/O 같이 대기하는 시간이 오래걸리는 task의 경우 CPU 활용률이 떨어질 수 있기 때문에, 컨텍스트 스위칭을 통해 효율적인 CPU 이용을 가능해집니다. 
    - 컨텍스트 스위칭은 인터럽트가 발생하면, 현재 프로세스의 정보를 PCB에 저장하고, 다음에 실행될 프로세스의 PCB에서 레지스터 상태, 프로그램 카운터, 스택 포인터 등을 읽어와 CPU에 복원합니다. 그 뒤, 다음 프로세스가 진행됩니다.
    - 이러한 컨텍스트 스위칭은 오버헤드가 발생하기 쉽기 때문에, 효율적인 스케쥴링과 멀티 스레딩 기법을 통해 최소한의 컨텍스트 스위칭이 이루어지게 주의를 기울여야 합니다. 
- PCB에는 어떤 정보들이 있고, 어떻게 사용되나요? 
    - PCB에는 다양한 정보가 들어가는데, 주로 프로세스의 상태, 프로그램 카운터, 레지스터 상태, 프로세스 우선순위와 같은 정보들이 들어가있습니다. 
- 커널 모드와 사용자 모드의 차이는 무엇이며, 왜 필요한가요?
    - 모든 응용 프로그램이 CPU와 같은 시스템 자원에 무분별하게 접근할 수 있게 된다면, 하나의 task에서 조금만 실수해도 컴퓨터 전체에 악영향이 갈 수 있는데, 그러한 문제를 막기 위해 커널 모드를 통해서만 시스템 자원에 접근할 수 있게 되어있습니다. 
    - 사용자 모드에서 실행되는 응용 프로그램은 운영체제의 서비스를 제공받을 수 없기 때문에 시스템 자원을 이용할 수 없으며,
    - 커널 모드에서 실행되는 운영 체제는 시스템 자원에 자유롭게 접근이 가능합니다.

    - (작성중)

<h3>프로세스 및 스레드</h3>

- 프로세스와 스레드의 차이점은 무엇인가요?
    - 프로세스는 기본적으로 독립된 메모리 영역을 할당받고, 스레드는 한 프로세스 내에서 공통의 메모리 영역을 공유하며 각 스레드는 스택 영역만을 따로 할당받습니다.
    - 이러한 특성으로, 프로세스는 IPC를 통한 프로세스간 통신을 하지 않는 이상 직접적인 메모리 공유가 어려운데 반해, 스레드는 하나의 프로세스 내에서 공유하는 메모리 영역이 있기 때문에 데이터 공유가 쉽습니다. 
    - 또한, 프로세스는 독립된 실행 단위이므로 각 프로세스는 독립된 생명주기를 가지는데 반해, 스레드는 한 프로세스 내에서 여러개의 스레드가 실행되기 때문에 프로세스의 생명주기에 종속됩니다.
    - 그리고, 프로세스는 말했듯이, 독립된 메모리 영역을 가지기 때문에, 컨텍스트 스위칭 시 더 많은 오버헤드를 야기하며, IPC를 통한 통신 비용도 큽니다. 이에 반해, 스레드는 프로세스간 컨텍스트 스위칭 보다는 적은 오버헤드를 가집니다.
- 멀티스레딩과 멀티프로세싱의 차이점은 무엇인가요? 각각의 장점과 한계점을 예시로 설명해주세요.
    - 두 개 모두 태스크의 병렬처리를 위해 사용되는 기법입니다.
    - 멀티스레딩은 하나의 프로세스 내에서 여러 스레드가 동작하는 방식이며, 스레드간 데이터 공유가 간편합니다. 장점으로는, 자원을 공유하기 때문에 효율적으로 데이터를 교환할 수 있고, 가볍습니다. 하지만, 파이썬의 경우 GIL(Global Interpreter Lock)으로 인해 한 번에 하나의 스레드만이 파이썬 객체에 접근할 수 있습니다. 또한, 여러 스레드가 하나의 자원 혹은 데이터에 동시에 접근할 경우, 동기화 문제가 일어날 수 있습니다. 
        -  파이썬 GIL : 하나의 스레드만이 하나의 객체에 접근 가능하기 때문에, 병렬 처리의 극한의 효율을 보기 어려움
    - 멀티프로세싱은 각각의 프로세스가 독립적으로 실행되는 것으로, 각 프로세스는 독립된 메모리 영역을 가지며, 서로 영향을 주지 않습니다. 별도의 메모리 영역을 가지기 때문에, 데이터 충돌이 없습니다. 또한 코어가 여러개일 경우 충분히 성능을 향상시킬 여유가 있습니다. 하지만, 프로세스간 통신은 상대적으로 느리며, 오버헤드가 발생할 가능성이 큽니다. 
- IPC란 무엇인가요? 또한, IPC가 필요한 이유는 무엇인가요?
    - IPC란, 프로세스들간의 통신방법으로, 메시지 전달 방식의 익명 파이프, 네임드 파이프, 메시지 큐, 소켓 등이 있으며, 메모리 공유 방식이 있습니다. 
        - 메시지 공유 : 파이썬에서 `import multiprocessing shared_memory = multiprocessing.Value('i', 0)`을 통해 하나의 공유 메모리를 생성하고, 멀티프로세싱을 할당할 함수들에 args로 `shared_memory` 전달.
        ```python
        import multiprocessing

        # 공유 메모리 생성
        shared_memory = multiprocessing.Value('i', 0)

        def increment(shared_value):
            for _ in range(1000000):
                with shared_value.get_lock():
                    shared_value.value += 1

        # 멀티프로세싱을 사용하여 increment 함수를 동시에 실행
        process1 = multiprocessing.Process(target=increment, args=(shared_memory,))
        process2 = multiprocessing.Process(target=increment, args=(shared_memory,))

        process1.start()
        process2.start()

        process1.join()
        process2.join()

        print("Counter:", shared_memory.value)
        ``` 
- 프로세스 간 통신(IPC) 방법으로 파이프와 메시지 큐의 차이는 무엇인가요?
    - 파이프는 기본적으로 단방향 통신을 지원하며, 주로 부모 프로세스와 자식 프로세스간 통신에 사용되며, 보통 간단한 구조입니다. 
    - 메시지 큐는 양방향 통신을 지원하며, 비동기적으로 실행할 수 있습니다.
- 교착상태(데드락)란 무엇인가요? 어떤 조건이 필요한가요? 또한 해결 전략에는 어떤 것들이 있나요?
    - 데드락은 동시성 문제 중 하나로, 복수의 프로세스나 스레드가 서로의 자원을 기다리며 무한히 대기하는 상태를 말합니다. 
    - 교착상태가 발생하기 위해서는 `1. 상호 배제, 2. 점유와 대기 3. 비선점 4. 원형 대기`라는 4가지 조건이 필요합니다. 
    - 이를 해결하기위해서는 **예방, 회피, 검출 후 회복** 등의 방법이 있습니다. 
    - **예방**은 4가지 조건 중 1가지라도 만족하지 않도록 하는 방법인데, `1. 상호 배제`의 경우, 모든 자원을 공유할 수는 없으며, `2. 점유와 대기`의 경우, 특정 프로세스에 모든 자원을 할당하는 방법인데 자원의 활용률을 낮춰버리며, `3. 비선점`의 경우, 선점 가능한 자원이 있는 반면, 아닌 자원도 있으며, `4. 대기 순환`의 경우, 자원에 번호를 붙이고, 오름차순으로 할당하는 할 수 있으나, 자원 번호 배분 방식에 따라 자원 활용률이 달라질 수 있습니다.
    - 다음으로, **회피**는 안전상태를 유지하도록 자원을 할당하는 방식으로, 스레드의 대기 여부를 결정하기 위해 현재 가용 자원, 각 스레드에 할당된 자원, 앞으로 요청하거나 방출할 자원 등의 추가적인 정보를 종합하여 교착상태가 일어나지 않도록 안전상태의 스레드들만 실행하고 불안전상태의 스레드는 대기 상태로 두는 방식입니다. 대표적으로, 은행원 알고리즘 등의 알고리즘을 이용하는 방법이 있습니다.
        - (참고 : https://howudong.tistory.com/366)
    - 마지막으로, **검출 후 회복**에는 `선점을 통한 회복`(교착 상태가 회복될 때까지 한 프로세스씩 자원을 몰아주는 방식)과 `프로세스 강제 종료를 통한 회복`(교착상태에 놓은 프로세스를 모두 강제 종료하거나 교착상태가 회복될때까지 한 프로세스씩 강제종료)이 있습니다.  
- 프로세스의 상태 전이에 대해 설명해주세요.
    - (작성중)
- 운영체제에서의 사용자 레벨 스레드와 커널 레벨 스레드의 차이점은 무엇인가요?
    - 사용자 레벨 스레드는 사용자 애플리케이션에서 관리되며, 커널은 이러한 스레드의 존재를 알지 못합니다. 이로 인해 컨텍스트 스위칭이 빠르지만, 한 스레드가 블로킹 상태가 되면 전체 프로세스가 블로킹 상태가 됩니다.
    - 커널 레벨 스레드는 운영체제 커널에서 직접 관리됩니다. 커널 레벨 스레드는 각 스레드가 별도로 관리되므로 하나의 스레드가 블로킹 상태가 되더라도 다른 스레드가 계속 실행될 수 있습니다. 그러나 컨텍스트 스위칭에는 추가적인 오버헤드가 발생합니다.
- 운영체제에서의 교착상태와 기아상태의 차이점은 무엇인가요?
    - 두 개 모두 서로 상대방의 자원을 기다리며 무한히 대기하는 상태를 의미하지만, 교착상태는 4가지 발생 조건이 동시에 충족될 때 발생하지만, 기아상태는 자원 할당이 불규칙하게 이루어져 특정 프로세스나 스레드가 지속적으로 자원을 얻지 못 할 때 발생합니다. 
    - 이에 따라, 해결방법이 다른데, 교착상태는 예방, 회복, 검출 후 회복 등이 있으며, 기아상태는 주로 공정한 자원 할당, 우선순위 스케쥴링 등이 있습니다.

<h3>메모리 및 파일 시스템</h3>

- 메모리 영역의 구조에 대해 설명해주세요.
    - 메모리 영역은 크게, `Code, Data, Heap, Stack`영역으로 구성되며, `Code`영역에서는 실행할 기계어로 된 코드가 저장되어 있습니다. 또한, `Data`영역에는 전역 변수나 정적 변수 등 한 번 읽고 끝낼 게 아니라 계속 유지하며 변경될 가능성이 있는 데이터가 저장됩니다. `Heap`영역에는 사용자가 프로그램을 실행할 때, 동적으로 할당한 데이터, 예를 들어 객체나 배열들이 저장됩니다. 그리고 `Stack`영역에는 지역변수나 함수의 매개변수 등 함수 호출 시 생성되는 데이터가 저장됩니다. 
- 힙과 스택영역에 저장되는 변수에 대한 설명을 코드를 예시로 들어 설명해주세요.
    - 파이썬은 메모리를 동적으로 관리할 수 없지만, 예를 들어 아래와 같이 `Heap`영역에 저장되는 데이터와 `Stack`영역에 저장되는 데이터를 구분할 수 있습니다. 
    ```python
    # 리스트나 클래스는 힙에 저장됨
    my_list = [1, 2, 3]
    class MyClass:
        def __init__(self, value):
            self.value = value

    my_instance = MyClass(42)

    # 함수의 호출과 관련된 지역 변수들은 스택에 저장됨
    def example_function():
        x = 10

    # 함수 호출 시 스택에 새로운 프레임이 생성되고 x가 저장됨
    example_function()
    ```
- 운영체제에서의 메모리 관리 전략에는 어떤 것들이 있나요?
    - 크게 `연속 메모리 할당(단일, 분할)과 페이징, 세그멘테이션, 가상 메모리` 등이 있습니다.
    - **연속 메모리 할당**의 경우, 프로세스는 하나의 연속된 메모리 블록을 할당받는데, 빠른 할당이 가능하나, 외부 단편화 문제가 일어날 수 있습니다.
    -** 페이징 기법**은, 논리적인 메모리 공간과 물리적인 메모리 공간을 일정 단위로 자르고, 페이지를 프레임에 할당하는 기법입니다.
    - **세그멘테이션**은 페이징 기법과 모두 비슷하나, 일정 단위가 아니라 가변 단위로 세그멘트나 프레임을 분할합니다.
    - **가상 메모리**는 스와핑을 통해 당장 사용하지 않는 프로세스를 디스크로 보내고 필요한 데이터를 메모리로 올리는 방법입니다. 
- 가상 메모리란 무엇인가요? 또한, 페이지 교체 알고리즘에 대해 설명해주세요.
    - 물리적 메모리보다 더 큰 프로그램이 실행될 수 있도록 컴퓨터 시스템에서 사용하는 메모리 관리 기법으로, 실제 메모리보다 더 큰 메모리를 필요로 하는 프로세스를 실행 가능하게 만듭니다. 
    - 페이지 교체 알고리즘에는 대표적으로 OPT, FIFO, LRU, LFU, MFU, NUR, Clock 등이 있습니다. (참고 : https://doh-an.tistory.com/28)
- 페이지와 세그먼트의 차이점은 무엇인가요?
    - 페이지는 고정 크기이고, 세그먼트는 가변 크기입니다. 따라서, 세그먼트가 외부 단편화가 일어날 가능성이 더 적습니다. 
- 페이지 폴트(Page Fault)가 발생하는 상황과 처리 과정을 설명해주세요.
    - 페이지 폴트는 프로세스가 실행 중인 동안 요청한 페이지가 물리적 메모리에 없어서 디스크로부터 해당 페이지를 가져와야 할 때 발생하는 현상을 말합니다. 즉, 필요한 페이지가 현재 물리 메모리에 로드되어 있지 않은 상태에서 접근되는 경우를 가리킵니다.
    - 페이지 폴트는 처음 메모리를 할당 받아야 하거나, 물리 메모리에 제거되어 디스크에 저장되어 있는 경우 발생합니다. 
    - 만약, 페이지 폴트가 발생되면, 운영체제가 페이지 폴트 예외를 발생시키고, 
    - 페이지 테이블을 검색 한 뒤, 처음이라 없으면 새로 할당하고, 페이지 교체 알고리즘에 따라 페이지를 스와핑 합니다. 
    - 그런 뒤, 페이지 폴트가 발생한 페이지를 물리 메모리에 로드한 뒤, 페이지 테이블을 업데이트하고, 프로세스를 계속 실행시킵니다.

<h3>동기화 및 스케줄링</h3>

- 프로세스(스레드) 동기화에 대해 설명해주세요.
    - 프로세스나 스레는 공유된 자원에 동시에 접근할 때가 있는데, 그 때 실행의 순서나 행동을 일치시켜 충돌을 막아야 합니다. 따라서 하나의 자원을 한 순간에 하나의 프로세스만이 이용하도록 제어해야 하며, 이러한 것을 프로세스의 동기화라고 합니다. 
    - 프로세스 동기화의 대표적인 기법으로는 세마포어, 뮤텍스, 모니터 등이 있습니다. 
- 임계영역이란 무엇인가요?
    - 동시에 실행하면 문제가 발생하는 자원에 접근하는 코드 영역을 의미합니다. 임계영역에 동시에 접근하면 자원의 일관성이 깨지는 현상을 레이스 컨디션(race condition)이라고 합니다. 
    - 운영체제가 임계영역 문제를 해결하기 위해서는 크게 `1. 상호 배제, 2. 진행, 3. 유한 대기`와 같은 원칙을 지켜야 합니다.
    - **상호 배제**는 한 프로세스가 임계영역에 접근했다면 다른 프로세스는 들어올 수 없다는 원칙
    - **진행**은 임계영역이 비어있다면, 들어오고자 하는 프로세스는 접근가능해야 한다는 원칙
    - **유한 대기**는 임계영역에 들어오고자 하는 프로세스는 언젠간 반드시 들어갈 수 있어야한다는 원칙 
- 세마포어와 뮤텍스의 차이점은 무엇인가요?
    - **뮤텍스 알고리즘**은 임계영역에 접근할 수 있는 프로세스나 스레드를 1개로 한정하고, `lock`과 `unlock`을 통해 동기화를 구현합니다. 
    - **세마포어 알고리즘**은 세마포어라는 임계영역에 들어갈 수 있는 프로세스의 수를 의미하는 정수 변수를 설정하고, `wait`과 `signal`함수를 통해 세마포어가 꽉 찼으면, wait을 호출하고, 들어가있던 프로세스가 끝나면 signal 함수를 통해 세마포어 값을 증가 시키고 wait 함수에 걸려있던 프로세스를 자원에 접근하게 합니다. 
    - **뮤텍스 알고리즘**은 단일 리소스에 대한 접근을 제어할 때나 데드락이 일어날 위험이 낮은 경우에 주로 사용됩니다. 이에 반해, **세마포어 알고리즘**은 카운터(세마포어)가 있기 때문에 여러 리소스에 대한 접근을 제어할 때나 데드락이 좀 더 빈번하게 일어날 것으로 예상될 때 사용됩니다.  
- 뮤텍스와 세마포어를 선택할 때 어떤 상황에서 어떤 것을 사용해야 하는지 설명해주세요.
    - 단일 리소스 + 데드락 위험 낮음 + 간단한 구현 => 뮤텍스
    - 다중 리소스 + 데드락 위험 높음 + 복잡한 구현 => 세마포어
- CPU 스케줄링 알고리즘에 대해 설명해주세요.
    - CPU 스케줄링은 여러 프로세스 중 어떤 프로세스를 먼저 실행시킬까를 정하는 알고리즘으로, 다양한 
- 프로세스 스케줄링에서 우선순위 스케줄링에 대해 설명해주세요.
    - 1. 선입선출 (First-Come-First-Served, FCFS)
    - 2. 최단 작업 우선 (Shortest Job Next, SJN 또는 Shortest Job First, SJF)
    - 3. 라운드 로빈 (Round Robin, RR)
    - 4. 최소 잔여 시간 우선 스케쥴링(Shortest Remaining Time Scheudling)
    - 5. 우선 순위 스케쥴링(priority Scheduling)
    - 6. 다단계 큐 스케쥴링(Multilevel Queue Scheduling)
- 스케줄러의 선점형(Preemptive)과 비선점형(Non-preemptive)의 차이는 무엇인가요?
    - 선점형 스케줄링은, 현재 실행 중인 프로세스를 다른 프로세스로 강제로 대체할 수 있는 스케줄링 방식입니다. 타임 슬라이스(시간 할당량)가 지나거나, 우선순위가 더 높은 프로세스가 도착하는 등의 이유로 현재 실행 중인 프로세스가 중단되고, 다른 프로세스가 실행될 수 있습니다.
    - 비선점형 스케줄링은 한 번 시작된 프로세스가 완료되거나, 대기 상태에 들어가거나, 수동으로 종료될 때까지 다른 프로세스로 대체되지 않는 스케줄링 방식입니다. 실행 중인 프로세스가 끝나거나 I/O 요청 등으로 대기 상태로 들어가야만 다른 프로세스가 실행될 수 있습니다.

<h3>기타 주제</h3>

- 인터럽트란 무엇인가요? 종류에는 어떤 것들이 있나요?
    - 인터럽트는 CPU가 프로그램을 실행 중에 예기치 않은 상황이나 외부 이벤트에 응답하기 위해 현재 수행 중인 작업을 일시 중단하고 그에 대한 처리를 하는 메커니즘입니다.
    - 크게 `1. 하드웨어 인터럽트, 2. 소프트웨어 인터럽트, 3. 내부 인터럽트, 4. 인터럽트 벡터 테이블` 등의 종류가 있습니다. 
- 소프트웨어 인터럽트와 하드웨어 인터럽트의 차이는 무엇인가요?
    - 소프트웨어 인터럽트는 시스템 콜이나 예외와 같은 상황에서 발생하는 인터럽트입니다.
    - 하드웨어 인터럽트는 외부 장치의 이벤트나 정해진 주기마다 발생하여 운영체제가 스케줄링 작업을 실행할 때 발생하는 타이머 인터럽트 등이 있습니다. 
    - 주로, 소프트웨어 인터럽트는 프로세스 내에서 명시하나, 하드웨어 인터럽트는 하드웨어 자체에서 일어나는 사용자의 제어에 의하지 않게 발생합니다.
